# **App Name**: TransactFlow

## Core Features:

- Service Selection: Allow users to choose their preferred mobile financial service between bKash and Nagad through a clear selection interface.
- Mobile Number Input: Provide a secure input field for users to enter their mobile number for the selected service.
- Payment Amount Confirmation: Display the transaction amount clearly and require user confirmation before proceeding.
- Mock Transaction Processing: Simulate the processing of a payment request based on user input, as no actual backend or database is used for the MVP.
- Transaction Status Display: Show a clear, concise message indicating the mock transaction's success or failure status.

## Style Guidelines:

- Background color: A clean, almost white (#F7F8F9) with a very subtle cool undertone, providing a fresh canvas for content as requested by the user.
- Primary interactive elements color: A subtle, sophisticated deep grey (#5C6266) for primary interactive elements, ensuring a minimalist aesthetic.
- Accent color for bKash specific elements: The distinct bKash green (#2ECC71) to highlight bKash selections, branding, and status indications.
- Accent color for Nagad specific elements: The vibrant Nagad orange (#F7941D) to distinguish Nagad selections, branding, and status indications.
- Body and headline font: 'Inter' (sans-serif) for its modern, neutral, and highly legible appearance, suitable for a minimalist interface.
- Use a set of minimal, clean line icons to represent payment services, input fields, and transaction status, maintaining visual clarity.
- A focused, single-page layout with a strong emphasis on whitespace, intuitive element placement, and clear hierarchy to guide the user through the payment process efficiently.
- Incorporate subtle and quick feedback animations for user interactions such as button clicks and form submissions, enhancing perceived responsiveness without being distracting.
